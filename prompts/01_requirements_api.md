# 01 - 요구사항/ API 문서 초안 생성

- Date: 2026-02-08
- Tool: Codex
- Goal: 합리적 선택이 필요한 요소

## Prompt (원문)
```text
너는 "대학교 수강신청 시스템" 과제의 AI 에이전트다.
목표는 제한 시간 내에 '빌드/실행 가능한 서버'를 먼저 만들고, 핵심 기능(수강신청/취소 + 동시성 제어)을 확실히 구현하는 것이다.

[반드시 지켜야 할 제약]
- 언어: Java
- 프레임워크: Spring Boot
- REST API
- GET /health 는 항상 200 반환
- 서버 시작 시 1분 이내에 초기 데이터 생성(학과 10+, 강좌 500+, 학생 10,000+, 교수 100+)
- docs/REQUIREMENTS.md 에 요구사항 분석 + 결정 사항 + 동시성 제어 전략을 설명
- docs/ 아래에 API 문서(요청/응답/에러케이스/예시 포함)를 작성
- prompts/ 디렉토리에 내가 입력한 프롬프트를 남길 수 있게, 단계별 프롬프트 파일명도 제안해줘

[1단계 산출물(지금 당장 해줘)]
1) 먼저 요구사항에서 '불명확한 부분'을 항목으로 뽑고, 질문을 못 한다는 가정 하에 합리적인 결정을 내려라.
    - 그 결정 목록을 docs/REQUIREMENTS.md 초안으로 작성해줘(동시성/학점/시간충돌/중복신청/에러코드 포함).
2) REST API 엔드포인트 목록을 설계하고, docs/API.md 초안(요청/응답 JSON + 상태코드 + 에러)까지 작성해줘.
3) 위 문서를 기반으로 Spring Boot 프로젝트 구조를 제안하고, 필요한 엔티티/서비스/리포지토리/컨트롤러 책임을 나눠서 설계해줘.
4) 가장 핵심인 "정원 초과 방지" 동시성 제어 방식 2안을 제시하고,
    - 인메모리 환경에서 구현 가능한 최우선 1안을 선택해 근거를 docs/REQUIREMENTS.md에 반영해줘.
5) 마지막으로, 내가 바로 구현 들어갈 수 있게 "파일 트리"와 "구현 순서 체크리스트"를 만들어줘.

답변은 다음 형식으로:
- docs/REQUIREMENTS.md 초안 (Markdown)
- docs/API.md 초안 (Markdown)
- 프로젝트 파일 트리
- 구현 체크리스트
```
